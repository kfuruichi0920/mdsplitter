# 作業指示書

## 0. 基本
* 思考は英語、回答は日本語とすること。
* 管理者に操作依頼が必要な場合は、待たずに、即依頼すること。

## 1. 作業プロトコル
* 作業指示が不明、曖昧であれば質問すること。

* 作業記録を残すこと(journal/journal_yymmdd.txt)。ただし個別に作業結果を出力する指示があれば無視してOK。
* 作業記録には、作業内容、決定事項およびその根拠を、不足なく示すこと。
* 根拠には、ファイルパスと行番号／ADR番号／APIスキーマ等を示すこと。

* 作業終了時、終了時点のコードベース対する詳細設計情報を更新すること(doc/詳細設計.md)
* 詳細設計の内容は
	- フォルダ、ファイル構成
	- フォルダの説明、設定ファイル個別の説明
	- ユースケース一覧
	- 処理の流れをPlantuml形式で出力（Activity図、Sequence図、State図)
	- ライブラリ利用していればその内容
	- 現状のコードベースで、未作成部分が識別可能な出力とすること
